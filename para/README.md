# 标定数据
## 总体说明
标定数据反映了典型的多传感器融合（相机和LiDAR）的内外参标定结果，包括了：相机内参，畸变系数，外参旋转矩阵，平移向量等方面。
其中：
1. 相机内参3*3矩阵写在intrinsics.txt文件
2. 相机畸变系数1*4矩阵写在dist_coeffs.txt文件
3. 旋转和平移外参4*4矩阵写在extrinsics.txt文件

## 详细说明
1. 相机内参矩阵（Camera Intrinsic Matrix）
camera_intrinsic:
[1453.72,   0.0,     1172.18,
   0.0,   1453.28,  1041.78,
      0.0,      0.0,      1.0]
该矩阵将相机坐标系下的3D点投影到像素坐标系（单位：像素）。
[ fx,  S, cx ]
[ 0, fy, cy ]
[ 0,  0,  1 ]

焦距
fx = 1453.72，fy = 1453.28，两者非常接近，说明像素在x和y方向上的物理尺寸几乎相同（正方形像素），且焦距尺度合理（通常对应中等视场角镜头）。
相机内参矩阵中的焦距定义如下：
  fx = F * (1 / dx)
  fy = F * (1 / dy)
  其中：
  F：实际物理焦距（单位：毫米）
  dx：每个像素在水平方向对应的物理尺寸（毫米/像素）
  dy：每个像素在垂直方向对应的物理尺寸（毫米/像素）
  因此，fx 和 fy 的最终单位是 （毫米）/（毫米/像素）= 像素。
  
主点（光心投影）
Hikvision CA-050-11UC RGB camera的最大分辨率为2448 × 2048
cx = 1172.18，cy = 1041.78，即图像中心附近。根据经验，图像分辨率很可能是 2448 × 2048 或类似尺寸（例如 2448×2048 时中心为 1224×1024，与此接近）。实际分辨率可由主点乘以2估算：2*cx ≈ 2344，2*cy ≈ 2083，因此分辨率为 2344×2083 或四舍五入后的常见工业相机尺寸（如 2448×2048 略有差异，可能因图像裁剪或标定误差导致）。

无倾斜
第一行第二列元素S（剪切因子）为0，表示像素坐标系两轴正交，无剪切变形。

2. 相机畸变系数（Distortion Coefficients）
camera_dist_coeffs: [-0.1210, 0.1113, 0.0016, 0.00013, -0.062353]
采用OpenCV的5参数径向‑切向畸变模型（顺序：k1, k2, p1, p2, k3）。
切向径向主要是针对于光线经镜头折射后最终成像在CMOS传感器上的位置偏移来定义的，其参考系是传感器平面（像平面） 上的二维坐标系。

径向畸变
k1 = -0.1210（负值 → 桶形畸变），边向外凸，角向内凹，——常见于广角/短焦镜头。
k2 = 0.1113（正值 → 枕形畸变），边向内凹，角向外凸，——常见于长焦/远摄镜头。
k3 = -0.062353（高阶项）。
组合效果：靠近图像边缘时，径向畸变由负转正，总体畸变曲线呈现“先桶后枕”的复合形状，常见于大视场镜头。

切向畸变
p1 = 0.0016，p2 = 0.00013，数值很小，说明镜头光轴与传感器平面的对齐精度较高，切向畸变可忽略。

畸变矫正公式
归一化平面坐标 (x,y) → 畸变后 (x_distorted, y_distorted)：
r² = x² + y²
radial = 1 + k1*r² + k2*r⁴ + k3*r⁶
x_distorted = x*radial + 2*p1*x*y + p2*(r² + 2*x²)
y_distorted = y*radial + p1*(r² + 2*y²) + 2*p2*x*y
- 实际应用时，通常调用OpenCV的 cv::undistortPoints() 或 cv::projectPoints() 自动处理。

3. 外参旋转矩阵（Camera Extrinsic Rotation Matrix）
camera_ext_R:[ 0.00298088, -0.999728,  -0.0231416, -0.00504636,  0.0231263, -0.99972,  0.999983,    0.00309683, -0.00497605]
该矩阵定义了从LiDAR坐标系到相机坐标系的旋转，满足：
P_cam = R × P_lidar + t
其中 R 是正交旋转矩阵（行列式 ≈ +1，验证：det(R) ≈ 0.999999，精度良好）。

几何解释
将 R 的行向量解释为相机坐标轴在LiDAR坐标系中的方向：
- 相机 X 轴（向右） 在LiDAR坐标系中为 R[0,:] ≈ [0.003, -0.9997, -0.023]
→ 几乎完全指向 LiDAR 的 负 Y 方向（因为 -0.9997 ≈ -1）。
- 相机 Y 轴（向下） 在LiDAR坐标系中为 R[1,:] ≈ [-0.005, 0.023, -0.9997]
→ 几乎完全指向 LiDAR 的 负 Z 方向（-0.9997 ≈ -1）。
- 相机 Z 轴（光轴向前） 在LiDAR坐标系中为 R[2,:] ≈ [0.99998, 0.0031, -0.0050]
→ 几乎完全指向 LiDAR 的 正 X 方向（0.99998 ≈ 1）。

结论：
- 相机的光轴（前方向）对齐 LiDAR 的 X 轴正向；
- 相机的右方向对齐 LiDAR 的 Y 轴负向；
- 相机的下方向对齐 LiDAR 的 Z 轴负向。
这相当于将LiDAR坐标系绕某个轴旋转约 90°（更精确说是三个90°的组合）。常见的LiDAR坐标系定义（如Velodyne）：X向前、Y向左、Z向上；相机坐标系（OpenCV）：X向右、Y向下、Z向前。上述旋转关系恰好将两个坐标系对齐：
- LiDAR的X（前）→ 相机的Z（前）
- LiDAR的Y（左）→ 相机的-X（右的反方向）
- LiDAR的Z（上）→ 相机的-Y（下的反方向）
因此该旋转矩阵正确描述了两种传感器之间的朝向关系。

欧拉角（近似）
采用 ZYX 顺序（偏航‑俯仰‑滚转）估算：
- 滚转（绕X轴）：atan2(R[2,1], R[2,2]) ≈ atan2(0.0031, -0.0050) ≈ -0.56°
- 俯仰（绕Y轴）：arcsin(-R[2,0]) = arcsin(-0.99998) ≈ -88.6°（接近 -90°）
- 偏航（绕Z轴）：atan2(R[1,0], R[0,0]) ≈ atan2(-0.00505, 0.00298) ≈ -59.6°
由于俯仰角接近 ±90°，欧拉角会出现奇异性，建议直接用轴角或四元数表达。但上述数值足以说明旋转幅度很大，并非微小调整。

4. 外参平移向量（Translation Vector）
camera_ext_t: [0.0025563, 0.0567484, -0.0512149]
该向量 t 表示 LiDAR 坐标系原点在相机坐标系中的坐标（单位：米），满足：
P_cam = R × P_lidar + t
- t_x = 0.0026 m（约 2.6 mm）
- t_y = 0.0567 m（约 5.7 cm）
- t_z = -0.0512 m（约 -5.1 cm）

含义：
将 LiDAR 的原点（通常位于雷达几何中心）变换到相机坐标系中，位于相机前方（Z轴方向）约 -5.1 cm？注意：t_z = -0.0512 m 表示在相机Z轴的负方向，即相机的 后方（因为相机Z正向为前方）。结合旋转关系：LiDAR的X正向对应相机Z正向，所以 LiDAR 原点到相机原点的平移向量在相机坐标系下为 (0.0026, 0.0568, -0.0512)。
若换到 LiDAR 坐标系中表示，则平移向量（相机原点相对LiDAR原点的位置）为 t_lidar = -R^T × t ≈ ?，计算得 t_lidar ≈ [-0.0512, -0.0026, -0.0568] 量级。
总体而言，两个传感器原点的空间偏移在 5 cm 以内，属于常见的安装偏差。

# 将LiDAR点云投影到图像的完整步骤：
假设 LiDAR 点坐标为 P_lidar = (Xl, Yl, Zl)，相机图像为 (u, v)。从激光雷达点云坐标投影到图像的步骤就是上述四项标定步骤的逆序。

1. 转换到相机坐标系
P_cam = R × P_lidar + t

2. 归一化投影（忽略畸变时）
x_n = P_cam.x / P_cam.zy_n = P_cam.y / P_cam.z

3. 应用畸变模型（OpenCV标准）
r² = x_n² + y_n²
radial = 1 + k1*r² + k2*r⁴ + k3*r⁶
x_d = x_n*radial + 2*p1*x_n*y_n + p2*(r² + 2*x_n²)
y_d = y_n*radial + p1*(r² + 2*y_n²) + 2*p2*x_n*y_n

4. 内参投影到像素
u = fx * x_d + cx
v = fy * y_d + cy
